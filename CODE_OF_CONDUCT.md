# Code of Conduct

Evidoc contributors should keep technical discussion specific, evidence-based, and respectful.

## Expected Behavior

- Explain tradeoffs with concrete evidence.
- Keep review comments actionable and scoped.
- Assume documentation drift can be subtle and unintentional.
- Protect private repository content and credentials.

## Unacceptable Behavior

- Harassment, personal attacks, or discriminatory language.
- Posting secrets, private code, or exploit details in public threads.
- Pressuring maintainers to bypass review, test, or safety gates.

Maintainers may edit, hide, lock, or remove contributions that violate these expectations.
